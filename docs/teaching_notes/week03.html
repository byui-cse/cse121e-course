Week 3

Week 03 Goals

  1. Introduce modules
  2. Introduce the structure of Erlang functions, including clauses, etc.
  3. Introduce guards.
  4. Introduce Erlang list processing.
  5. Introduce the Erlang syntax for map, filter, and reduce.

Grading and Feedback

  • Since there are no graded assignments in this course, you will need to be proactive in reviewing and providing constructive feedback on students' tasks. (See the General Teaching Notes.) Check the results from the W02 Quiz with the Tasks to determine if any students who completed the quiz did not turn in a task submission. If students do not submit their tasks but receive points from the quiz, you should reach out to them to ask for the submission so you can check their understanding. You are allowed to remove their quiz points if they do not respond to you or refuse to submit the task for review and feedback.

Sample W04 Announcement

Week 03 Wrap Up - Release Friday

Dear CSE 121E Students,

First of all, make sure to take a look at the sample solution for the weekly task after you finish it so you can see another approach. There is a video included to walk through the sample solution, so you can use that to help you understand if you get stuck.

Over the last week we've learned how to create and use functions in Erlang. The way we've been writing functions works great for single-threaded sequential programs. However, one of the most powerful features of Erlang is its support for writing concurrent programs. This next week we will begin learning the Erlang way of writing code that can easily spawn multiple processes, making concurrent programs relatively easy to write.

If you are not familiar with the concept of spawning a process or writing concurrent code in another language, I would encourage you to do a little research before reading about how to do it in Erlang. You might try using Google to search for topics such as "what is concurrency" and "parallelism vs concurrency".

You could also try using a generative AI tool such as ChatGPT to learn about concurrency. Try prompting ChatGPT with something like "I am a student who is learning to program in erlang. What does concurrency mean and why would I want to use erlang to write concurrent programs?"

Then read the response and ask additional questions to help you dig deeper into the topic.

I want to let you all know that I am here to help you if you get stuck or have questions. Most of the groups have been doing an outstanding job of meeting together, working on the task together, and helping one another with questions. I've been very impressed with how well you have been using your groups to help each other. Keep up the great dialog with one another!

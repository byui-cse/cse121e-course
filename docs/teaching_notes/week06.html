CSE 121e: Teacher's Notes

Week 6

Week 06 Goals

  1. Students will think of a problem they would like to solve using the Erlang language they have learned.
  2. Students will write the code to solve the problem they have proposed.

This week does not include any new content. Students will spend their time working on a final project of their choosing. Their project should take about 3 hours. Your announcement and discussions with them could include ideas for final projects or invitations to them to reach out to you if they would like to get feedback on their final project ideas.

Some project ideas include:

  • Revisit an assignment from a previous programming class and complete it in Erlang.
  • Create a program that solves an interesting problem or a problem in the student's discipline.
  • File I/O: Build an application that can read data from a file, process it, and then write the results to a new file. This could be a log analyzer, a data transformer, or a simple database.
  • Explore passing messages between distributed systems: Create a simple system where multiple Erlang nodes can send and receive messages. This could simulate a microservices architecture or a basic chat system between distributed systems.
  • A to-do list application: Develop a command-line to-do list application where users can add, view, and delete tasks. The tasks can be persisted in a file or an in-memory data structure.
  • A chat application using Erlang processes: Develop a simple chat server where multiple clients can connect and send messages to each other using Erlang's message passing and concurrency features. Focus on creating processes for each client and handling message passing.
  • Task Scheduler System: Implement a system that schedules and executes tasks (like sending emails, generating reports, etc.) at specified intervals. This project can leverage Erlang's time-handling and process management capabilities.
  • Basic Key-Value Store: Create a simple in-memory key-value store where you can set a value for a key and retrieve it later. This project will help you understand state management in Erlang processes and the concept of message passing.
  • File Monitoring System: Build a system that monitors changes in a specific directory (like new files added, files deleted, etc.) and logs these events. This project can utilize Erlang's file handling and process monitoring features.
  • Simple Web Server: Develop a basic web server that can handle HTTP requests and serve static content. This will give you exposure to network programming in Erlang and its standard libraries for handling HTTP.
  • Data Processing Pipeline: Create a pipeline that processes data (like logs or transaction records) through multiple stages (filter, transform, aggregate) using Erlang processes. This project can demonstrate Erlang's capability in handling parallel data processing tasks.
  • Online Voting System: Implement a simple system for online voting, where users can vote for options, and the system tallies the votes. This project can help you understand how to handle user inputs, state management, and concurrency.
  • IoT Device Simulator: Simulate an IoT (Internet of Things) environment where multiple devices (like sensors) send data to a central server. This project will allow you to explore Erlang's ability to handle numerous concurrent connections and real-time data processing.

Before the Week Begins

Inspire Your Students

  • Share an insight from a scripture or a general conference talk.
    • You can be a great role model for students about what it means to study the gospel and apply it to your own life. During this lesson, consider making a video or sending a class-wide email highlighting an insight from a general conference talk or the scriptures and how it applies to your life.
  • Consider sharing a gospel insight on how you have applied gospel principles in your career.
  • Discuss what you think are the most difficult part of the final project or share ideas for an appropriate final project.
  • Consider sharing an example of how you used concepts from this week in your own profession. Perhaps the experience can be related to a programming challenge you or a team had to resolve.

During the Week

  • Monitor the Microsoft Teams channels. Interacting with students is where much of your time should be spent. You may consider reaching out to those students who appear not to be regularly participating.
  • Create and make available one or more 10 minute videos relative to the goals of this module.
  • Reach out to any students in the class that appear to have 'gone dark.'

Grading and Feedback

  • Since there are no graded assignments in this course, you will need to be proactive in reviewing and providing constructive feedback on students' tasks. (See the General Teaching Notes.) Check the results from the W05 Quiz with the Tasks to determine if any students who completed the quiz did not turn in a task submission. If students do not submit their tasks but receive points from the quiz, you should reach out to them to ask for the submission so you can check their understanding. You are allowed to remove their quiz points if they do not respond to you or refuse to submit the task for review and feedback.
  • For the final project, the students only need to spend about 3 hours. This means the projects will be relatively simple. Offer encouragement and support to the students. As long as they complete something, even if it is simple, they will be able to claim credit for this assignment.

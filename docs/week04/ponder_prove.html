CSE 121e: Week 04

Tasks

Your Task

Description

What you want to accomplish with this task is to solve the design problems listed in the reading for this week so you can run a bunch of computations concurrently without one crash taking everything down. Also, you will use some of the Erlang programming principles learned earlier in the class to write easily read, understood, and, therefore, easily maintained code.

Before beginning this Task, make sure you have reviewed the concurrent programming course content.

The Process

Complete the following steps

  1. Create a process that provides factorial for its functionality. The process must keep track of the last number it was sent and the factorial value calculated from that number and use these two values to minimize the computation required for the next time the factorial process is used. If it is not possible to re-use these values to reduce the required computation, then do as must be done.
  2. In the same module as the factorial function, write a start/0 function that spawns the factorial function as a process and returns the factorial process' PID.
  3. In the same module as the factorial function, write a client function that sends an integer to the factorial process and waits for a response.

The Testing Driver

Complete the following steps

  1. Create a separate module containing a testing driver function, driver/0, that uses the spawned factorial process and calls client function you've written. It must call the testing driver function described 1000 times. Make it interesting by asking for the factorial of numbers that are not too small but not too large either.
  2. From within the REPL, run driver/0 and observe the results.

Review

After completing the previous steps:

  1. Review one of the many possible solutions for this task,
  2. Compare and contrast your code with the possible solutions’ code, and
  3. Make note of any improvements you could make for future tasks.

Stretch (optional)

Dealing with large numbers can be slow when applying factorial. Write a function that can accept a maximum and minimum number. This function should have as its value the product of all the integers in the range [min,max]. Decide how many processes running this range-product should be spawned for a number based on the number's size.

Write a function that spawns an appropriate number of processes, uses them to calculate factorial for a large number, and then shuts them down.

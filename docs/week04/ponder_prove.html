CSE 121e: Week 05

Ponder and Prove

Your Task

Description

What you want to accomplish with this task is to solve the design problems listed in the reading for this week so you can run a bunch of computations concurrently without one crash taking everything down. Also, you will use some of the Erlang programming principles learned earlier in the class to write easily read, understood, and, therefore, easily maintained code.

Before beginning this Task, make sure you have reviewed the gen_servers and supervisors course content.

The Process

Complete the following steps

  1. Create a process that provides factorial for its functionallity. The process must keep track of the last number it was sent and the factorial value calculated from that number and use these two values to minimize the computation required for the next time the factorial process is used. If it is not possible to re-use these values to reduce the required computation, then do as must be done.

The Testing Driver

Complete the following steps

  1. Create a seperate module containing a testing driver function that spawns a process calling the client function of the factorial module's client function. This new function is responsible for makeing the call, waiting for the response, and then printing the response out to the console.
  2. Write an additional function, driver/0, as part of this module that calls the testing driver function described in the previous item 1000 times. Make it interesting by asking for the factorial of numbers that are not too small but not too large either.
  3. From within the REPL, run driver/0 and observe the results.

Review

After completing the previous steps:

  1. Review one of the many possible solutions for this task,
  2. Compare and contrast your code with the possible solutions’ code, and
  3. Make note of any improvements you could make for future tasks.

Stretch (optional)

Apply your googling skills to see some ideas about how you can use a supervisor to supervise many running instances of the same gen_server module.

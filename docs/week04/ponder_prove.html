Tasks

Your Task

Description

What you want to accomplish with this task is to solve a design problem. In the reading example, you can run a bunch of computations in a process, but if one request causes a crash, it takes everything down. This means that everyone else using that process for computations has a request fail. This is not good. As part of this redesign, you will want to use the Erlang programming principles learned earlier in the class to write easily read, understood, and, therefore, easily maintained code.

Before beginning this Task, make sure you have reviewed the concurrent programming course content.

The Process

Complete the following steps

  1. Create a module that has a process providing factorial for its functionality.
  2. In the same module as the factorial function, write a start/0 function that spawns the factorial function as a process and returns the factorial process' PID.
  3. In the same module as the factorial function, write a client function that sends an integer to the factorial process and waits for a response.
  4. Divide up the simple_calc module into three distinct modules, each with one of the calculation behaviors. Include in each module a client function similar to the one from simple_calc.

The Testing Driver

Complete the following steps

  1. Create a separate module containing a testing driver function, driver/0, that spawns at least two processes for each of the four modules created earlier. The driver/0 function must use the modules' client functions. It must call each client function 1000 times. Make it interesting by asking for the results from interesting numbers. Also include sending unrecognizable messages to each of the for process types.
  2. From within the REPL, run driver/0 and observe the results.

Review

After completing the previous steps:

  1. Review one of the many possible solutions for this task,
  2. Compare and contrast your code with the possible solutions’ code, and
  3. Make note of any improvements you could make for future tasks.

Stretch (optional)

  1. Create a modified version of your solution so that the factorial process runs in one instance of the erl REPL and the client in a separate erl REPL instance on one machine.
  2. Explore how to run the client on one machine and the factorial process on another. Create a modified version of you code to do so.

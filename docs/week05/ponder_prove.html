CSE 121e: Week 05

Tasks

Your Task

Description

You have so many friends that you have a hard time keeping track of them. 😉 Create a stateful process that tracks your amazing and abundant friends.

Before beginning this Task, make sure you have reviewed the stateful processes course content.

The Process

Complete the following steps

  1. Create a module that has a run/1 function that tracks the names of your friends.
  2. In the same module, write a start/0 function that spawns the run/1 function as a process and returns the the process' PID.
  3. In the same module as the run/1 function, write a client function that sends friend status change messages to the process and waits for a response.
  4. Write the run/1 function in such a way that, based on which message it receives, a friend can be added, removed, a check can be done to see if someone is your friend, or a list of your current friends can be retrieved.
  5. Make sure that unrecognized messages are also dealt with in such a way that the process doesn't crash.

The Testing Driver

Complete the following steps

  1. Create a separate module containing a testing driver function, driver/0, that spawns at least two processes for the module and uses the modules' client functions. The testing driver function must make multiple calls to the rpc function. Make the driver interesting by adding, removing, and checking friends. Add at least 1,000 friends. Also include sending unrecognizable messages to the process.
  2. From within the REPL, run driver/0 and observe the results.

Review

After completing the previous steps:

  1. Review one of the many possible solutions for this task,
  2. Compare and contrast your code with the possible solutions’ code, and
  3. Make note of any improvements you could make for future tasks.

Stretch (optional)

  1. Create a modified version of your solution so that the friend process runs in one instance of the erl REPL and the client in a separate erl REPL instance on the same machine.
  2. Explore how to run the client on one machine and the spawned process on another. Create a modified version of you code to do so.

CSE 121e: Week 05

Ponder and Prove

Your Task

Description

What you want to accomplish with this task is to solve the design problems listed in the reading for this week so you can run a bunch of computations concurrently without one crash taking everything down. Also, you will use some of the Erlang programming principles learned earlier in the class to write easily read, understood, and, therefore, easily maintained code.

Before beginning this Task, make sure you have reviewed the gen_servers and supervisors course content.

The gen_servers

Complete the following steps

  1. Separate the add, multiply, and divide functionalities of the calcs module into three gen_server modules, one module per functionality.
  2. Create modules for calculating a binomial coefficient, factorial, and generating all permutations of any list. Again, one module per functionality.
  3. The factorial gen_server should use the same ideas for reducing computation that you used for last week's task.

The Supervisor

Complete the following steps

  1. Write a supervisor called calcs_supervisor that supervises all of these gen_servers using the one_for_one supervision technique.
  2. From within the REPL, run calcs_supervisor and exercise each of the gen_servers.
  3. Fix any bugs you find.

The Testing Driver

Complete the following steps

  1. Create a module that contians one function per gen_server. Have each of these function spawn a call to the client function of the gen_server it is responsible for executing, and then wait for the response and print it out to the console.
  2. Write an additional function, driver/0, as part of this module that calls each of the functions described in the previous item. Make sure that you make a call to divide that causes a crash after you have sent each of the list-type functions a list of 100,000,000 numbers.
  3. From within the REPL, run driver/0 and observe the results.

After you have a working solution, you should compare what you created with another possible solution.

Stretch (optional)

Apply your googling skills to see some ideas about how you can use a supervisor to supervise many running instances of the same gen_server module.

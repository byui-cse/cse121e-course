Your Task

Explore I/0

Since you will be using output to the erl REPL as a major way to debug your Erlang code in this class, It's a good idea to be able to retrieve data from and diplay data to the REPL. Since you have litte to no experience with Erlang, there is a web page that you can use as an example for for the code you're writing. Don't bother reading the question asked, just look at the answer given. That's where the example code is. You will also want to take a look at an ascii table that shows you that characters are actually numbers in computing systems.

To this end, create a folder using your OS called task01. Load this folder in VSCode and in it, create a file called io_explore.erl. In the file,

  1. declare a module called io_explore (file names and module names must match in Erlang),
  2. export two functions (in one export statement). Call the functions
    • print_greeting_nice (with no parameters and asks the user for their name) and
    • print_greeting_strange (with no parameters and asks the user for their name)
  3. write the print_greeting_nice function using the format string "It is so nice to meet you ~s.~n" as the first parameter to the io:format function,
  4. load, compile, and debug the module,
  5. write the print_greeting_strange function using the format string "It is so nice to meet you ~w.~n" as the first parameter to the io:format function, and
  6. load, compile, and debug the module with these new changes.

There are many things you haven't learned yet regarding this code you'll write. You'll learn about them later in the class. Don't be overly concerned about not understanding everything right now. Do, however, consider what was printed out by print_greeting_strange taking into account what you see in the asci table.

When calling the functions in the erl REPL, remember to put quotation marks around the name you type in and to end the line with a period. If you don't do these two things, you will get some errors you aren't ready to tackle yet.

AFTER you have completed and debugged your code, take a look at one of many possible solutions and compare it to the code you wrote.
